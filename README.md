# Электронная библиотека

Автоматизированная информационная система «Электронная библиотека» для ведения реестра книг и читателей, осуществления поиска книг и оставления рецензий.

## Технологии

- Python 3 + Flask
- SQLite
- Jinja2
- Bootstrap 5
- EasyMDE (редактор Markdown)

## Запуск проекта

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Инициализировать базу данных (создаст файл `library.db` с тестовыми данными):

```bash
python setup_db.py
```

3. Запустить приложение:

```bash
python app.py
```

4. Открыть в браузере: `http://localhost:5000`

## Возможности пользователей

### Неаутентифицированный пользователь
- Просмотр главной страницы со списком книг
- Просмотр информации о книге
- Вход в систему

### Пользователь
- Все возможности неаутентифицированного пользователя
- Написание рецензий на книги (одна рецензия на книгу)
- Просмотр своих рецензий

### Модератор
- Все возможности пользователя
- Редактирование данных книг

### Администратор
- Все возможности модератора
- Добавление новых книг с обложками
- Удаление книг
- Просмотр статистики (журнал действий, статистика просмотров)
- Экспорт данных в CSV

## Основной функционал

- **Список книг** — пагинация (10 книг на странице), сортировка по году, отображение рейтинга и количества рецензий
- **Просмотр книги** — обложка, описание (Markdown), жанры, рецензии пользователей
- **Добавление/редактирование книги** — форма с мультиселектом жанров, загрузкой обложки, редактором Markdown
- **Рецензии** — оценка от 0 до 5, текст с поддержкой Markdown
- **Удаление книги** — модальное окно подтверждения, каскадное удаление связанных записей
- **Обложки** — дедупликация по MD5-хэшу, хранение в файловой системе

## Вариант 4: Учёт истории посещений

- **Популярные книги** — 5 наиболее просматриваемых книг за последние 3 месяца (только аутентифицированные пользователи)
- **Недавно просмотренные** — 5 последних просмотренных книг (включая неаутентифицированных)
- **Лимит посещений** — не более 10 посещений страницы одной книги в день на пользователя
- **Статистика** (для администратора):
  - Журнал действий пользователей с пагинацией
  - Статистика просмотров с фильтром по дате
  - Экспорт в CSV

## Работа с базой данных

База данных SQLite хранится в файле `library.db` в корне проекта.

### Просмотр через Python

```python
import sqlite3
conn = sqlite3.connect('library.db')
c = conn.cursor()
c.execute('SELECT * FROM books')
for row in c.fetchall():
    print(row)
conn.close()
```

### Просмотр через консоль

Если установлен `sqlite3`:

```bash
sqlite3 library.db
.tables                  # список таблиц
SELECT * FROM books;     # все книги
SELECT * FROM users;     # все пользователи
.quit                    # выход
```

### Пересоздание базы

```bash
python setup_db.py
```

### Структура таблиц

- `roles` — роли пользователей (Администратор, Модератор, Пользователь)
- `users` — пользователи (логин, пароль, ФИО, роль)
- `genres` — жанры книг
- `books` — книги (название, описание, год, издательство, автор, объём)
- `book_genres` — связь книги-жанры (многие ко многим)
- `covers` — обложки книг (файл, MIME-тип, MD5-хэш)
- `reviews` — рецензии (книга, пользователь, оценка, текст, дата)
- `book_visits` — история посещений (книга, пользователь, дата)
